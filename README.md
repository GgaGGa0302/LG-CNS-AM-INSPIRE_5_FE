Issues 탭 기본 사용법
1. 새 이슈 만들기 (New issue)
저장소 상단의 Issues 탭에 들어가서 우측 상단의 초록색 New issue 버튼을 누릅니다.
Title (제목): 해야 할 일이나 발생한 문제점을 명확하게 적습니다.
예: 메인 화면 로그인 버튼 UI 구현, 비밀번호 입력 시 앱이 튕기는 버그 수정
Description (본문): 구체적으로 어떤 작업이 필요한지 상세히 기록합니다. 체크박스(- [ ] 할 일 목록)를 만들어 세부 목표를 쪼개 적어두면 매우 유용합니다.

2. 우측 보조 메뉴 활용하기
글 작성 화면 오른쪽에 있는 메뉴들을 설정하면 관리가 훨씬 체계적으로 변합니다.
Assignees (담당자): 이 작업을 누가 할 것인지 지정합니다. 본인을 포함한 팀원 4명 중 해당 코드를 짤 사람을 지정해 두면 "아, 이 기능은 00 님이 하고 있구나" 하고 바로 알 수 있습니다.
Labels (라벨): 이 글의 성격을 색깔 태그로 분류합니다. bug(버그), enhancement(기능 추가), documentation(문서 작성) 등의 라벨을 붙여두면 나중에 목록만 봐도 프로젝트 진행 상황을 파악하기 좋습니다.

💡 [꿀팁] 커밋(Commit)과 이슈 자동 연결하기
이슈를 작성하면 제목 옆에 #1, #2처럼 고유한 이슈 번호가 자동으로 부여됩니다. 이 번호를 커밋할 때 써먹으면 아주 편리합니다.
예를 들어 #3번 이슈로 '회원가입 기능 개발'이 등록되어 있다고 가정해 보겠습니다.
작업을 끝낸 뒤, 코드를 커밋(Commit)할 때 메시지 창에 Fixes #3 또는 Resolves #3 이라고 적어서 푸시(Push)해 보세요.
코드가 깃허브에 올라가는 순간, 깃허브가 스스로 이를 인식해서 열려있던 3번 이슈를 "작업 완료(Closed)" 상태로 자동으로 닫아버립니다!
